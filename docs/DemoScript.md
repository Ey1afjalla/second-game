# 演示脚本

目标：让您在 10 分钟内看到完整闭环。

## 0-1 分钟：项目定位

这是一个面向游戏运营和 AI Agent 工程岗位的作品集。它不是单纯做一个小游戏，而是把玩家行为、运营指标、Agent 建议、配置热更新和桌面交付串成一个最小 LiveOps 闭环。

展示：

1. 打开 README 的 3 分钟亮点。
2. 快速说明四个主视图：游戏、运营、Agent、配置。

## 1-3 分钟：小游戏核心循环

操作：

1. 进入游戏页。
2. 补货一个 NPC 偏好商品。
3. 点击接待顾客。
4. 展示金币、收入、满意度、库存和消息变化。

游戏循环故意保持轻量，但每个关键动作都会进入事件流，后续运营后台和 Agent 都读取这些真实事件。

## 3-5 分钟：运营诊断后台

操作：

1. 切到运营页。
2. 展示 KPI、趋势、任务完成率、商品转化、NPC 互动率、流失节点和 A/B 测试。

讲法：

这里模拟游戏运营岗位关心的问题：玩家为什么没有成交，哪些商品低转化，哪一步流失最严重，活动版本是否改善留存评分。

## 5-7 分钟：AI 运营 Agent

操作：

1. 切到 Agent 页。
2. 展示运营日报、异常原因假设、行动计划、调参建议和工具轨迹。

Agent 不直接改游戏状态，只读取运营证据并输出结构化建议。这样可以解释 Agent 的输入、输出和安全边界。

## 7-9 分钟：配置热更新

操作：

1. 切到配置页。
2. 展示 schema 校验结果。
3. 点击应用草案。
4. 回到游戏页接待顾客，展示首购补贴生效。
5. 回到配置页点击回滚。

AI 生成的活动配置必须经过校验和人工确认。应用后配置会真正影响玩法，回滚也会留下历史记录。

## 9-10 分钟：桌面交付

操作：

1. 说明 `npm run desktop:build` 可以生成 Windows 安装包。
2. 展示 Tauri 版本和安装包路径。

这个项目从 Web 原型演进到桌面应用，适合本地演示、内网工具和运营控制台类场景。
