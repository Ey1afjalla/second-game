# Agent 设计记录

## 当前版本

当前版本使用本地规则模拟 AI 运营 Agent，暂未调用外部模型。

原因：

1. 第一版重点验证游戏运营闭环。
2. 避免在未建立安全边界前让模型直接影响游戏数值。
3. 便于后续把规则输出替换为真实 LLM 调用。

## Agent 输入

1. 游戏天数。
2. 事件埋点。
3. 商品配置。
4. 接待人数、成交数、收入、流失次数、转化率。

## Agent 输出

1. 运营摘要。
2. 数据证据。
3. 风险判断。
4. 运营建议。
5. 活动配置草案。

## 安全边界

1. Agent 只能生成建议和配置草案。
2. Agent 不允许直接修改金币、声望、满意度等关键数值。
3. 后续真实模型输出必须通过 schema 校验。
4. 配置进入游戏前必须人工确认。
